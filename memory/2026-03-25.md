## Rain World Environment Research
- Answered Ethan's question about how Rain World handles environment art
- Key findings: Levels are hand-authored geometry with procedural surface treatment on top (materials, erosion, slime effects). Pre-rendered to PNGs. Three depth layers for parallax. NOT procedurally generated levels.
- Five editor layers: Geometry → Tiles/Materials → Effects → Light → Props
- Relevance for Genesys: our tile system = geometry editor. Missing: material system (procedural fills), effect overlays (erosion/moss), baked lighting. All deferrable — grey boxes work for now.

### Design Decisions
- **No passive battery drain** — gamers hate it. Drain-on-use + environmental drain zones model instead
- **Battery costs**: Tech dash 0.5%, Grapple 0.5%, Shield 1.0%, EVE L3 scan 5.0%. Battery cell pickup restores 20%
- **Suit integrity absorption**: `SuitIntegrity / 200` = fraction absorbed. 100% suit → 50% damage reduction. Minimum 1 damage always passes through
- **SuitIntegrity and Battery live on Player class** (not Game1) so TakeDamage() can access them
- **Starting values**: SuitIntegrity=31f, Battery=80f (post-crash state)
- **BreakableBattery tile (ID 94)**: drops battery cell, mirrors Breakable(70) which drops hearts
- **Tech degradation below 15% suit**: 30% chance tech dash fizzles (CancelDash + orange sparks)
- **Cipher overheat**: deferred to later
- Deleted `docs/first-30-minutes.md` — working from `game-design/master-task-list-0-30.md` instead (commit `cac35f6`)
