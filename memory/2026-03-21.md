### Scan System Redesign (~00:52–01:46 UTC)

**Research:** Reddit r/Metroid (Prime scanning complaints), YouTube CgA6XOuaT7Y ("Metroid Prime Had A Scan-tastic Story")

**Major design decisions (Ethan-confirmed):**
- **NO scanning live hostile enemies.** Scan corpses, sleeping/caged/trapped creatures. Creates trapping/camouflage/observation gameplay loops.
- **~10 scan entries per EVERYTHING.** Depth from different CONTEXTS not repeat presses.
- **Player NEVER learns native language.** EVE translates. Communicate via gestures/pictograms. Fetch quests = deduction.
- **Scan button = L** (no finger conflict with WASD)
- **Combat knowledge from combat** (kill counts → telegraphs → weak points). Scan = strategic.
- **EVE combat alerts** scale with scan depth: "Above you!" for Wingbeater dives
- **Knowledge Tree** branches: Ecology, Geology, Archaeology, Linguistics, Biology, Engineering

**Docs:** `game-design/scan-system.md` (full design), GDD updated, `game-design/ecosystem.md`

### Tile Grid Origin Bug — ROOT CAUSE FIX (commit a1ad320)
- **The bug**: When tile grid was expanded to origin (-32,-400), ALL physics code still divided world coords by tileSize without subtracting origin. Every tile lookup was reading the WRONG cell.
- **Caused**: enemies falling through floors, slide stuck bug (HasHeadroom found phantom solid tiles), SnapToSurface placing enemies wrong
- **Fixed in**: `EnemyPhysics.cs` (rewrote with WorldToTileX/Y helpers), `Player.cs HasHeadroom`
- This was the real cause of the "slide into wall = stuck crouching" bug — HasHeadroom couldn't find clearance because it was checking wrong tiles

#### Slope Outline Saga (~5 iterations)
- Full box around slopes → stray lines in empty triangle space
- Diagonal only → no grid definition between adjacent slopes
- Full grid + diagonal → outline around empty space (user: "that's what we DON'T want")
- Solid-edge-only → stray verticals on tall side
- **Final (commit 6e8f4b7)**: outlines traced from actual DrawSlopeTile fill geometry row-by-row
  - Each slope type's diagonal edge computed from the same math as the fill
  - Straight edges only where fill touches tile boundary (e.g. gentle floor has vertical on solid side from halfway down)
  - Shaved tiles: diagonal on shaved corner, straight edges on remaining 3 solid sides (partial where shave cuts in)
- **Key lesson**: never approximate slope outlines — always derive from the exact fill computation
