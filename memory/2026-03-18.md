## Session 3: Slope Collision Overhaul (~04:05-05:37 UTC)

### Bugs Fixed (multiple iterations)
- **Ceiling slope tiles inverted**: Drawing code had SlopeCeilRight/Left producing same shape as floor slopes. Fixed by swapping fill direction (solid above diagonal). Later swapped Left/Right labels to match collision math.
- **Can't walk up 45° slope from flat tile**: Solid floor merged rects were blocking horizontal movement. Added adjacent-slope-tile detection to skip horizontal push-out near slopes. Loosened vertical proximity check in GetSlopeFloorY.
- **Slope→flat jitter**: Added slope sticking logic — probes below player when grounded+moving, speed-scaled probe depth (4-24px based on vel.X). Prevents bouncing at terrain transitions.
- **Slide cancelled at slope→flat transition**: Added _slideAirFrames grace period (6 frames) before cancelling slide when briefly airborne.
- **Left/right slope asymmetry**: Center-only sensor meant SlopeUpRight detected at bottom when entering from left. Fixed with 3-sensor system (center + left foot + right foot) in GetSlopeFloorY and GetSlopeCeilY, matching Sonic engine approach.
- **Teleporting when adjusting position on slope**: Removed leading-edge slope check that sampled wrong position. Multi-sensor system replaced it.
- **Ceiling bonk on solid tiles between ceiling slopes**: Added nearCeilSlope detection (±2 tiles wide) to skip hard ceiling rect collision when ceiling slopes are nearby.
- **Debug room not showing purple**: Level name was "Untitled", detection used filename. Fixed _editorSaveFile tracking on level load.
- **Garden level overwritten**: Restored from git commit 1885516.

### Research: Sonic Physics Guide (Sonic Retro via Wayback Machine)
- Tiles use height arrays (16 values per column), not math-computed slopes
- Independent sensors: A/B (floor), C/D (ceiling), E/F (push) — never fight each other
- Sensor extension/regression for smooth tile-to-tile transitions
- Push sensors disabled on slopes (prevents our exact clipping bug)
- Ground Speed variable (separate from X/Y Speed) preserves momentum across angle changes
- Slope Factor: gravity applied along slope surface for uphill slowdown/downhill speedup
- Applied concepts: multi-sensor collision, slope sticking, push-out suppression near slopes

## Session: Ceiling Slope Physics & Deflection (06:04-06:43 UTC)

### Ceiling Slope Deflection — Root Cause Found & Iterated
- **vel.X override bug**: Deflection was setting vel.X to 400+ but normal movement code (`vel.X = inputX * Speed`) ran next frame and zeroed it. Added `_ceilDeflectTimer` to preserve momentum temporarily.
- **Direction was wrong**: SlopeCeilRight was pushing right (into narrow side) instead of left (open side). Swapped.
- **Downward bounce**: `vel.Y = upSpeed * 0.3f` was proportional, causing strong downward push.
- After multiple iterations, Ethan wanted simple sliding, not launching/boosting.

### Final Ceiling Slope Design (IMPORTANT)
- **Position correction ONLY** — no velocity manipulation at all
- If `pos.Y < slopeCeilY`: push player to `slopeCeilY`. That's it. 6 lines.
- Sliding emerges naturally from jump vel + gravity + per-frame push-out
- Double jumps work because vel.Y is never zeroed by ceiling code
- Scales to any future slope angle — just define surface math
- Removed all deflection/boost/timer code
- Commits: `d7ce42f` → `12f433e` → `f01c566` → `208101e` → `eecc98a`
