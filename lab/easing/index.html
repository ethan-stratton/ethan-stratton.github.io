The most important range in game dev is 0 to 1. Damage falloff, alpha fades, UI animations, hit reactions, camera lerps, loot probability — everything maps to a number between zero and one.

Linear interpolation looks robotic. Easing functions warp the t parameter so movement feels organic. Don't modify the equation — modify the input.

Terminology from Squirrel Eiserloh's GDC talk: SmoothStart = slow start, fast end. SmoothStop = fast start, slow stop. Forget "ease in/out" — it's backwards from intuition.
